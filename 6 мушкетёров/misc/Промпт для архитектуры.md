Мы делаем сервис для сбора дата сетов для ИИ. В проекте участвуют заказчик и $n$-ое кол-во исполнителей. Заказчик загружается dataset (zip-file с $k$ обьектов) на сервис, который распределяется среди исполнителей, каждый исполнитель размечает $\frac{q\cdot k}{n}$ обьектов, где $q$ - кол-во независимых исполнителей, которым будет отправляться обьект для дополнительной валидации (1 исполнитель не может разметить 2 одинаковых обьекта). Сервис должен работать как онлайн (человек разметил → сразу отправилось), так и оффлайн (человек размечает кластер данных локально и, как появляется доступ к "сети" → данные синхронизируются). У заказчиков должна быть возможность интеграции своих серверных систем, например, для хранения данных, ради обеспечения максимальной приватности.

Исполнителей назначают сами заказчики, они не связаны с сервисом. Авторизация происходит через внешние сервисы (vk, yandex, etc), а далее уже авторизованные исполнитили могут зайти в комнату заказчика.
Логика room-ов:
Заказчик создает "комнату", к которой могут присоединяться исполнители по ссылке и вводу пароля. Любой исполнитель, имеющий ссылку и пароль может присоединиться к "комнате". Заказчик, в роли админа "комнаты", видит всех исполнителей и может их удалять из "комнаты". 

Логика работы исполнителей:
Каждый обьект из dataset отправляется на разметку q людям, после чего сервис принимает решение
