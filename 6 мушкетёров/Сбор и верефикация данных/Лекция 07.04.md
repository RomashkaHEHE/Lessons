Защита от парсинга и ддос атак:
- Rate limit (сервер скользящим окном считает количество запросов)
  Чтобы обойти используем переключение между айпишниками или снижаем темп. Сначала можно бить почаще потом случайно потом пореже. delay = $-\lambda \ln(1-R_{случайн}), \lambda - 1 < k$
- captcha (сервисы, библиотеки, браузеры)
  Используем сервисы, библиотеки, браузеры, в которых встроено прохождение капчи
  После прохождения нужно верифицировать что мы прошлы
- Поведенческий анализ (отслеживает блуждание курсора + чекает что это действительно браузер + динамику клавиатуры + динамику кликов)
  Используем библиотеки имитации человека
- HTML + CSS (обфускация html)
  Используем скраперы. 

Отличительные черты скрапера:
- Обладает собственной навигацией
- Эмулирует поведение человека
- Обрабатывает динамический контекст (JS и другие)
- Управляет паузами, сменой API, IP и может даже иметь прокси67

Типология скраперов:
- Однострочные (собирают данные с одной строчки url'а - вакансии, цены, поисковые выдачи)
- Пагинационный???? (проходит по графу страницы - каждая карточка выдачи)
- Паутинный (обходит весь сайт по внутренним ссылкам)
- Гибридный (где можно идём в API, где нельзя парсим)

Архитектура скрапера:
нарисуйте рисунок прикольный всё что через запятую то параллельно
Планировщик -> диспетчер очереди url -> менеджер айпишников, менеджер user agent, менеджер прокси, обработчик капчи
								   ->(из диспетчера очереди) загрузчик страниц -> парсер

	валидация, очистка -> загрузчик страниц

Популярные скраперы: 
- Selenium (+ эмулятор работы в браузере - очтяжёлый)
- Scrapy (+ асинхронность + паутина + прокси - проблемы с обработкой динамического контента)
- playwright (+ эмуляция кликов и скроллов - медленный - ресурсоёмкий)
- BeautifulSoup (+ простой + понятный - ломается при изменение структуры сайта)
